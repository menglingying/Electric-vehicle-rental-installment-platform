# 新增需求确认结果（已选定）

本文件为客户确认后的最终口径，作为开发实现依据。

## 1) 下单资料口径

- 全部必填、一次性提交，不做分步补录与草稿保存。
- 资料提交入口：`/api/h5/orders/{orderId}/kyc`
- 资料字段：
  - 身份证信息（已有）
  - 联系人：姓名、手机号、关系
  - 工作情况：就业状态（枚举）
  - 收入情况：月收入区间（单选）
  - 住家地址：省/市/区 + 详细地址

就业状态枚举（采用）：
- 在职 / 自由职业 / 个体经营 / 学生 / 待业 / 退休

月收入区间（采用）：
- 1000–2000
- 2001–3000
- 3001–4000
- 4001–5000
- 5001–8000
- 8001–12000
- 12001+

地址选择规则（采用）：
- 省/市/区三级联动选择，必须全部选择
- 详细地址必填（道路/门牌/小区等）

## 2) 商品分类层级

- 采用三级分类：大分类 → 系列 → 具体车型（SKU/商品）
- H5 先选大分类，再选系列，最后选择具体车型

## 3) 调价与合同规则

- 调价仅允许在“审核前”进行，审核通过后锁价。
- 调价触发：重新生成分期计划并留痕。
- 调价后合同处理：重签合同（作废旧合同，生成新合同并重新签署）。
