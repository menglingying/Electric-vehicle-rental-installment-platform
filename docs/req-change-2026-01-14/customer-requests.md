# 客户新增需求整理（2026-01-14）

本文件用于记录客户新增需求，作为一期范围的补充说明与评审依据。

## 1. 下单资料补充（H5）

现状：下单仅采集身份证信息。

新增：下单需补充以下资料字段（与身份证信息一起提交）。

- 联系人信息：联系人姓名、联系人手机号、关系
- 工作情况：就业状态、单位名称、岗位/职务、单位地址
- 收入情况：月收入区间或自定义金额（需明确口径）
- 住家地址：省/市/区 + 详细地址

建议交互：
- 作为下单流程的独立步骤或与身份证资料同页展示。
- 字段校验：手机号格式校验，必填项与可选项需明确。
- 就业状态、收入区间可走字典配置，便于后台维护。

## 2. 商品分类层级（H5 + 后台）

现状：商品列表为“全部型号明细”，无大类/系列层级。

新增：需要“三级分类”选择路径。

推荐结构：
- 大分类（例如：城市通勤/外卖专用/高性能等）
- 系列（在大分类下）
- 具体车型（最终可下单的 SKU/商品）

落地建议：
- 后台商品管理新增分类维护能力（分类树 + 排序 + 启停用）。
- H5 商品入口按“大分类 -> 系列 -> 车型”逐层选择。
- 商品列表与搜索需支持按分类过滤。

## 3. 下单后价格可调整（租期驱动）

诉求：客户下单后，运营可根据租期/周期差异调整租金价格。

建议规则：
- 允许在“订单审核前”调整租金/分期计划，审核通过后锁定。
- 调价触发：重新生成租金分期计划，并保留调整前后记录。
- 调价内容：租金单价、期数、还款周期变化需明确是否允许联动修改。
- 留痕：记录操作人、时间、调整原因，供审计与追溯。

交互参考：列表中展示“按期租金”差异（见需求截图示意）。

## 影响范围（需评估）

- H5：下单流程、表单校验、分类入口与展示。
- 后台：分类管理、订单调价入口、计划重算与审计记录。
- 服务端：订单资料字段扩展、分类模型与查询、租金调整逻辑与留痕。

## 待确认问题与可选方案

以下为与客户确认用的“选项清单”，每题可从中选择或组合。

### 1) 下单资料是否全部必填？是否允许分步补录？

选项：
- A. 全部必填、一次性提交（推荐）
  - 适用：风控要求高、减少后续补录流程。
- B. 核心必填 + 非核心可补录
  - 核心示例：身份证、联系人、住址、收入区间；工作情况可补录。
- C. 分步补录
  - 下单先填核心字段，审核前补齐全部资料。

建议补充：字段是否允许“保存草稿”，草稿保留时长。

### 2) 工作/收入字段枚举与取值口径

工作情况（示例枚举，可选其一口径）：
- A. 就业状态：在职 / 自由职业 / 个体经营 / 学生 / 待业 / 退休
- B. 职业类型：上班族 / 外卖骑手 / 配送员 / 制造业 / 服务业 / 其他

收入情况（区间示例，支持单选）：
- 1000–2000
- 2001–3000
- 3001–4000
- 4001–5000
- 5001–8000
- 8001–12000
- 12001+

口径说明（需确认）：
- 以“月收入”为口径（税前/税后需确定）。
- 是否允许“自定义金额”作为兜底。

### 3) 调价允许的时点

选项：
- A. 仅“审核前”可调价（推荐）
  - 通过后锁价，避免合同与计划反复变化。
- B. 审核前 + 已审核未签署可调
  - 需同步更新待签合同与分期计划。
- C. 已审核未生效可调（签署完成但未生效）
  - 需二次确认与审计记录。

### 4) 调价后的合同处理规则

选项：
- A. 重签合同（推荐）
  - 作废旧合同，生成新合同并重新签署。
- B. 补充协议
  - 原合同保留，新增“租金调整补充协议”。
- C. 仅内部调整（不改合同）
  - 仅适用于未签署或未生成合同的订单。

地址字段明确口径（给客户选择）：
- 地址必填：省/市/区三级联动 + 详细地址必填。
- 地址选择方式：选择省后自动加载市，选市后加载区，最后填写详细地址。
