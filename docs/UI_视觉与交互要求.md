# UI 视觉与交互要求（H5 + PC 后台）

本文件用于明确“长什么样、怎么交互、哪些地方不能自由发挥”，保证多人协作时 UI 统一、可验收。

相关文件：
- UI UX Pro Max 使用方式：`docs/UI_UX_Pro_Max_使用指南.md`
- 组件落地基线（Arco/Vant）：`docs/UI_UIX_Pro_Max_执行规范.md`

## 1) UI 选型（已定）

- **PC 后台**：Vue 3 + TypeScript + Vite + `Arco Design` + `ArcoDesign Pro（Vue）`
- **H5 客户端**：Vue 3 + TypeScript + Vite + `Vant 4`

说明：UI UX Pro Max 只用于给 AI 提供“风格/配色/字体/布局/UX 规则”的建议与提示词；实际页面必须用 Arco/Vant 组件落地。

## 2) 风格定位（本项目建议）

- **业务气质**：偏“金融分期/风控”场景，要求稳重、清晰、可信赖；避免花哨动效与过强装饰。
- **视觉风格**：现代简洁（Minimal / Clean），轻阴影、弱渐变可用但要克制；优先信息层级与可读性。
- **一致性原则**：同一类页面（列表/表单/详情/结果）只允许使用同一套组件组合与同一套间距/字号层级。

## 3) 设计 Token（统一来源）

硬性要求：
- 禁止在业务代码里写死自定义色值/阴影/圆角/字体（除非是布局适配）。
- 所有颜色/圆角/间距统一走主题 Token（后台以 Arco Token 为准，H5 做映射）。

建议基线（可按品牌色微调，但必须整体一致）：
- 主色（Primary）：用于主按钮/高亮/关键链接（建议继续用 Arco 默认主色体系）
- 状态色：Success/Warning/Danger 仅用于状态与结果提示，不滥用
- 圆角（Radius）：统一 4/6/8（不要混用奇怪值）
- 间距（Spacing）：统一 4/8/12/16/24/32（移动端优先 8 的倍数）

## 4) PC 后台（ArcoDesign Pro）UI 要求

- **布局**：统一采用 ArcoDesign Pro 的基础布局（侧边栏 + 顶栏 + 内容区），不自定义第二套框架布局。
- **列表页**：表格 + 筛选区 + 批量操作（如需）遵循 Pro 模式；筛选项默认折叠策略统一。
- **表单页**：统一字段栅格（1/2 列）、统一 label 对齐方式；必填/校验文案风格统一。
- **状态展示**：订单状态/逾期分层/审核状态统一用 Tag/Badge 组件（禁止用自定义彩色文字冒充状态）。
- **图表（如有）**：优先少量关键图（订单趋势、回款趋势、逾期分层），避免为了“好看”堆图表。

## 5) H5 客户端（Vant）UI 要求

- **适配**：移动端优先，兼容微信内置浏览器/Chrome/系统浏览器；安全区（safe-area）要处理。
- **主流程 CTA**：下单/确认/提交类按钮需要“固定底部提交栏”（Vant 组件优先），且有禁用态与加载态。
- **信息层级**：价格/期数/每期应还/首付（押金）等关键字段要突出，次要说明默认折叠或弱化。
- **弹窗与确认**：所有关键动作（签署、提交订单、支付跳转）需二次确认，并明确后果与可撤销性（如有）。

## 6) 交互与可用性（统一要求）

- **表单校验**：输入错误必须定位到字段并给出可执行的提示（不要“提交失败”这种泛提示）。
- **加载与空态**：列表加载骨架/空态/错误态统一样式；接口失败给重试入口。
- **权限与不可见**：后台无权限的菜单不展示或禁用（不要点进去才报错）；关键操作需二次确认。
- **可访问性（最低要求）**：按钮可点区域足够大；文字与背景对比度可读；状态色不要只依赖颜色区分（搭配文案/图标）。

## 7) 页面清单（用于对齐 UI 产出顺序）

后台优先：
- 登录
- 商品管理：列表/新增编辑/详情
- 订单管理：列表/详情/审核
- 还款管理：计划/记录/对账
- 逾期管理：分层列表（0-3/3-10/10-30/30+）

H5 优先：
- 商品列表/详情
- 下单与分期方案选择
- 订单列表/详情
- 还款计划/还款记录
