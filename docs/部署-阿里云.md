# 部署到阿里云（Ubuntu 22.04）

目标机器：`47.120.27.110`（root）

约束：服务器上已有一个服务在跑，端口大概率是 `5000`，本部署不会占用/修改 `5000`。

## 部署结果（默认端口）

- H5：`http://47.120.27.110:8088`
- 后台：`http://47.120.27.110:8089`
- API：仅在 Docker 内网使用（由 Nginx 反向代理 `/api`），默认不对公网暴露

## 你需要准备

- 本地可用的 `ssh` / `scp`（Windows 10/11 自带 OpenSSH 即可）
- 服务器允许 root SSH 登录（你现在就是 root 账号）
- 推荐：配置 SSH Key（避免每次输入密码）

## 一键部署（在你本机运行）

1. 确认你能连上服务器：
   - `ssh root@47.120.27.110`

2. 在项目根目录运行：
   - `powershell -ExecutionPolicy Bypass -File deploy\\aliyun\\deploy.ps1`

（只打包不上传：`powershell -ExecutionPolicy Bypass -File deploy\\aliyun\\deploy.ps1 -BundleOnly`）

## 用 Python 一键部署（推荐：可直接用密码登录）

如果你不想配置 SSH Key，可以用 Python 脚本直接用密码登录并完成部署（同样不占用/不影响 `5000` 端口）：

- 运行（会提示输入 SSH 密码 + MySQL 容器 root 密码）：
  - `python deploy/aliyun/deploy.py`

- 如果你的 SSH 密码保存在本地文件（例如你提到的 `E:\\电动车租赁分期平台\\服务器密码`），可用：
  - `python deploy/aliyun/deploy.py --ssh-password-file \"E:\\电动车租赁分期平台\\服务器密码\"`

- 只打包不上传：
  - `python deploy/aliyun/deploy.py --bundle-only`

3. 第一次运行会在服务器上生成 `/opt/evlease/.env`，请编辑并至少修改：
   - `MYSQL_ROOT_PASSWORD=...`

4. 修改后在服务器执行：
   - `bash /opt/evlease/remote/deploy.sh /opt/evlease`

## 放通端口（如果你服务器开了防火墙）

- `ufw allow 8088/tcp`
- `ufw allow 8089/tcp`

## 重要：阿里云安全组

如果你本机访问 `http://47.120.27.110:8088`/`8089` 超时，但服务器内 `curl http://127.0.0.1:8088` 是 200，通常是阿里云“安全组入方向”没放通端口。

在 ECS 控制台给实例绑定的安全组里添加入方向规则（TCP）：
- 8088（H5）
- 8089（后台）

## 说明

- `deploy/aliyun/docker-compose.yml` 会启动 4 个容器：`mysql`、`api`、`h5`、`admin`
- `h5`/`admin` 都通过 Nginx 代理 `/api/*` 到后端 Spring Boot
- 不会触碰你服务器上原有的 `5000` 端口服务
