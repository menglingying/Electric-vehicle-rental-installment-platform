# 分期计算逻辑（来自 `电动车分期计算H5` 现行实现）

本文档描述当前“分期/租赁计算器”的计算规则与返回字段含义，供 `电动车租赁分期平台` 复用或对齐实现。

## 1. 输入与期数限制

输入参数：
- `total_price`：车价总额（单位：元，必须 `> 0`）
- `periods`：期数（仅支持：`3`、`6`、`9`、`12`）

当 `periods` 不在支持列表中时，计算应直接报错（无效分期期数）。

## 2. 固定配置（现行）

来自 `E:\电动车分期计算H5\calculator.py`：

### 2.1 押金比例

- `DEPOSIT_RATE = 0.25`（押金 = 车价总额 × 25%）

> 注意：押金仅作为“首付/押金展示字段”，不参与每期还款的本金拆分计算。

### 2.2 月利率（按期数配置）

`INTEREST_RATES`：
- `3` 期：`0.06`（6%）
- `6` 期：`0.06`（6%）
- `9` 期：`0.05`（5%）
- `12` 期：`0.05`（5%）

## 3. 核心计算规则（现行）

### 3.1 关键中间量

1) 押金
- `deposit = total_price * DEPOSIT_RATE`

2) 剩余本金（展示用）
- `remaining_principal = total_price - deposit`

3) 月利率
- `interest_rate = INTEREST_RATES[periods]`

4) 每期本金（均分车价总额）
- `monthly_principal = total_price / periods`

5) 每期租金/利息（固定按“车价总额”计算）
- `monthly_interest = total_price * interest_rate`

> 这不是“等额本息/按剩余本金计息”的模型：现行逻辑的租金/利息每期固定不变，且与剩余本金无关。

6) 每期应还总额
- `monthly_payment = monthly_principal + monthly_interest`

7) 分期总还款（不含押金）
- `total_payment = monthly_payment * periods`

### 3.2 分期明细（details）

明细从第 1 期到第 `periods` 期逐期生成：
- 初始 `current_remaining = total_price`
- 每期结束后：
  - `remaining_after_payment = current_remaining - monthly_principal`
  - `current_remaining = remaining_after_payment`

每期明细字段：
- `period`：期数序号（从 1 开始）
- `principal`：本期本金（`monthly_principal`，保留 2 位小数）
- `interest`：本期租金/利息（`monthly_interest`，保留 2 位小数）
- `payment`：本期总额（`monthly_payment`，保留 2 位小数）
- `remaining`：剩余金额（`max(0, remaining_after_payment)`，保留 2 位小数）

## 4. 返回字段定义（现行）

返回对象包含：
- `total_price`：车价总额（2 位小数）
- `deposit`：押金（2 位小数）
- `remaining_principal`：剩余本金（展示用，2 位小数）
- `periods`：期数
- `interest_rate`：月利率（小数，如 `0.05`）
- `monthly_principal`：每期本金（2 位小数）
- `monthly_interest`：每期租金/利息（2 位小数）
- `monthly_payment`：每期应还（2 位小数）
- `total_payment`：分期总还款（不含押金，2 位小数）
- `details`：分期明细列表

## 5. 四舍五入与展示注意

现行实现对多个字段使用 `round(x, 2)`：
- `monthly_principal`、`monthly_interest`、`monthly_payment`、`total_payment`、`details[*].*` 都会保留 2 位小数。
- 由于浮点数与四舍五入，`total_payment` 可能出现 `6399.96` / `6400.00` 这类差异（取决于实现是“先精确计算再 round”还是“先 round 再累加”）。

平台实现建议：
- 统一使用“以分为单位的整数”或 `Decimal` 进行金额计算，最后再格式化输出，避免浮点误差。

## 6. 示例（`total_price = 4000`）

### 6.1 9 期（利率 5%）

- 押金：`4000 * 0.25 = 1000`
- 每期本金：`4000 / 9 = 444.44...`
- 每期租金/利息：`4000 * 0.05 = 200`
- 每期应还：`444.44... + 200 = 644.44...`
- 分期总还款（不含押金）：约 `5800`

### 6.2 12 期（利率 5%）

- 押金：`1000`
- 每期本金：`4000 / 12 = 333.33...`
- 每期租金/利息：`200`
- 每期应还：`533.33...`
- 分期总还款（不含押金）：约 `6400`

## 7. 现行 API 对接（参考）

来自 `E:\电动车分期计算H5\app.py`：

- `POST /api/calculate`
  - 请求 JSON：
    - `total_price`: number
    - `periods`: number
  - 响应 JSON：
    - `success`: boolean
    - `data`: 上述返回字段对象

## 8. 与业务对齐需确认的问题（建议）

如果平台要“正式化”该计算逻辑，建议在产品/风控/财务层面确认：
- 押金是否应为 25%？是否按车型/活动可变？
- “租金/利息”是否应该按剩余本金计息（递减）还是固定按车价总额？
- 押金是否计入总成本/总应付展示？现行 `total_payment` 不包含押金。
- 期数是否只有 3/6/9/12，未来是否需要扩展（如 18/24）。
