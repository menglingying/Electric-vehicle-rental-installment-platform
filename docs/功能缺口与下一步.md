# 功能缺口与下一步（对照 `docs/需求文档`）

说明：当前工程已跑通“后台上架商品 → H5 下单 → 人工审核 → 交付/取车/归还 → 还款标记 → 提醒/逾期 → 导出 CSV”，并预留“电子签/收银台/短信”接口口径（目前为模拟实现）。

## 一期已实现（可联调）

- H5：手机号登录（dev 固定验证码）、商品列表/详情、下单、订单列表/详情、还款计划/还款记录展示、电子签/收银台预留入口
- 后台：登录、商品上架（新增/编辑，多图上传+封面排序）、订单审核与履约流转、黑名单、还款管理（标记已还）、提醒与短信（模拟发送+记录）、逾期分层、CSV 导出（订单/还款/逾期/短信）、订单状态日志
- 服务端：对应 API 全部可用（Spring Boot 版本运行在 `8080`，Node 版本运行在 `8082`）

## 仍缺（建议按优先级）

### P0（影响“可上线/可交付”）
- 短信验证码：固定验证码仅用于测试；生产必须接入短信通道（含频控、黑名单、风控策略）
- 电子签真实对接：签署发起、回调验签、合同 PDF 存档与下载、幂等与重试
- 收银台真实对接：支付发起、回调验签/签名、自动入账/对账、幂等与异常处理

### 已完成（P0）
- 数据持久化：已接入 MySQL 8.0（商品/订单/还款/黑名单/日志/短信记录），并验证重启后数据仍在

### P1（影响“业务完整性/效率”）
- 订单状态更细化：是否需要“交付预约/取消/异常关闭原因”等字段与流程
- 字典配置：期数/周期/押金比例/押金分期方案等目前未做后台可配置（需求文档提到“后台可配置”）
- 二维码/链接：后台生成给客户的 H5 入口链接/二维码（需求文档业务流程里提到）
- 客户管理：客户列表/订单聚合视图（除黑名单外）

### P2（体验/运维/安全）
- RBAC 权限：目前只做了后台登录；还未做“管理员/运营/财务/催收”菜单与数据范围控制
- 操作审计：目前订单有状态日志，但未做全局审计（黑名单变更、配置变更等）
- 定时任务：提醒/逾期目前是“查询生成”，未做定时跑批与任务队列（后续接短信/收款后更重要）
- 文件与资源：已支持商品多图上传（`/uploads/*`，服务器磁盘持久化）并在 H5 商品详情轮播展示；合同 PDF 存储/下载权限控制（MinIO/OSS）仍缺
