## 1. 后台商品页（新增/编辑商品）改动

### 1.1 商品需要支持“配置/规格”选择（类似截图的属性表）

客户反馈：商品不是“单一配置+单一定价”，需要做成“可选配置”（类似截图中的表格：属性名称/名称/分期方式/价格），客户下单时先选配置再下单。

建议落地为“商品规格（SKU/配置项）”列表（至少先覆盖电池配置）：
- 配置项名称：例如「电池配置」
- 配置项可选值：例如「空车」「含电池」
- 分期方式：截图出现「租完归还」等字样，需要确认是否作为字段固化到订单/合同
- 该配置对应价格：至少支持配置到“租金/期（元）”；若客户口径为“价格/天”，需要明确与周期的换算关系

### 1.2 电池配置：空车/含电池 + 分别定价（按最新截图）

客户最新截图期望（后台新增商品弹窗内）：
- 电池配置需要提供两档价格输入（或两条配置项）：
  - `空车租金/期（元）`：必填（示例：677）
  - `含电池租金/期（元）`：可选（不填则使用默认租金；默认租金具体指向需确认）
- 客户在 H5 下单时可选择「空车 / 含电池」，系统据此使用对应租金生成还款计划。

需要确认的规则（避免实现偏差）：
- “默认租金”指什么：是商品原 `租金/期` 字段，还是 `空车租金/期`？
- 若“含电池租金/期”为空：是禁止选择“含电池”，还是允许选择但使用默认租金？
- 配置选择结果需要在：H5 订单详情、后台订单详情、导出 CSV、合同（如有）中固化展示。

## 2. H5 下单流程改动（商品详情 → 提交下单）

### 2.1 提交下单后：增加“补充资料（实名/KYC）”流程

客户反馈：当前在商品详情页提交下单后直接创建订单；希望在下单后补充身份证与个人情况信息，并完成基础影像采集。

期望补充的资料页（建议作为下单后的引导流程）：
1) 身份证信息
   - 身份证正面照片
   - 身份证反面照片
2) 人脸自拍照
   - 一张自拍照（用于人脸核验/留存）
3) 个人情况（字段示例）
   - 工作信息（职业/单位/工作城市等，具体字段待确认）
   - 居住信息（现居地址/居住时长等，具体字段待确认）
   - 联系人（至少 1 人；姓名/手机号/关系，数量与必填规则待确认）

需要确认的流程规则：
- 资料是“下单前必填”还是“下单后补填，但未完成前订单不可进入审核/签署/放款流程”？
- 资料未完成时，订单状态如何展示（例如：`待补充资料`）以及后台审核侧如何处理？
- 图片存储方式：本地 `uploads` / OSS / MinIO（现阶段可先按本地上传，后续替换）。

## 3. 下单参数与分期/押金规则补充

### 3.1 期数选择：客户可选择“多档期数”

客户反馈：下单时客户可以选“好多期”（不止当前固定值）。

落地要点：
- 期数可配置：由后台配置可选项（例如 `3/6/9/12/18/24`，具体清单待确认）。
- 前端展示：选择期数后自动刷新计算结果与还款计划。

### 3.2 租金自动生成：选择期数后自动生成还款计划

客户反馈：租金自动生成还款计划，并在下单页/确认页展示。

落地要点：
- 明细字段：每期应还金额、到期日/还款日、期次序号等。
- 展示口径需要与 `docs/分期计算逻辑.md` 或最新确认口径一致（避免“现行实现”与“需求口径”不一致）。

### 3.3 还款方式：需要让客户自行选择

客户反馈：还款方式用户可选（例如：自动扣款/手动转账/线下收款等，具体选项待确认）。

落地要点：
- 还款方式作为订单字段固化，并影响后续支付/催收/对账展示。
- 后台需可查看/导出。

### 3.4 押金：当选择“押金分期付款方式”时，按分期规则处理

客户反馈：押金如果选择分期付款方式，需要按既定分期计算逻辑处理（参考 `docs/分期计算逻辑.md`）。

需要进一步明确的计算与展示点：
- 押金是“单独收取（首付）”还是“参与分期计划（押金也拆分到多期）”？
- 押金与租金计划的关系：是否合并展示为同一还款计划，或分别展示两张计划？
- `docs/分期计算逻辑.md` 当前口径为“押金 25% + 每期本金均分 + 每期固定利息”，与本仓库现行实现（押金按比例抵扣最后几期）存在差异；需要选定最终口径并统一实现与文案。

## 4. 与截图不一致点（需要调整的页面行为）

### 4.1 H5 商品详情页：下单前必须先选“配置/规格”

客户反馈“第二个截图不对”的核心点：当前页面是“单一租金+静态电池文案+直接提交下单”；期望为：
- 商品展示区：电池不应只是静态文本，而应提供“配置选择”（至少：空车/含电池）
- 选择配置后：页面上的价格/租金与还款计划应随之变化
- 订单创建时：把选择的配置一并提交并写入订单

### 4.2 配置项的计价口径（租金/期 vs 价格/天）

截图出现「价格/天」列，但后台最新期望也出现「租金/期（元）」输入框。需要在需求层面确认统一口径：
- 若以 **价格/天** 为准：则 `租金/期 = 价格/天 × 周期(天)`，并明确取整/四舍五入规则
- 若以 **租金/期** 为准：则无需按天换算，但若周期可选（7/10/15/30天），不同周期下是否需要不同租金/期（否则会出现按天单价不一致）

## 5. 验收点（按客户反馈可验收）

- 商品新增/编辑：可配置“空车/含电池”两种电池选项，并分别配置租金；H5 下单可选择其一，订单中能看到选择结果。
- H5 下单：提交下单后进入“资料补充”流程，可上传身份证正反面 + 人脸自拍，并填写工作/住家/联系人信息。
- 下单期数：可选择多档期数；选择后自动生成并展示还款计划。
- 还款方式：客户可选并写入订单；后台可查看/导出。
- 押金分期：当选择押金分期方式时，按最终确认的规则生成押金计划/合并计划，并可在订单详情展示。
