# 下单资料字段与校验

## 目标
补充下单后的资料采集，除身份证与人脸外，新增联系人、工作情况、收入情况、住家地址。全部必填。

## 字段清单（必填）

### 1) 现有字段（保持不变）
- 身份证正反面照片
- 人脸自拍照
- 真实姓名、身份证号

### 2) 联系人信息
| 字段 | 类型 | 规则 | 说明 |
| --- | --- | --- | --- |
| contactName | string | 2-20 字 | 紧急联系人姓名 |
| contactPhone | string | 11 位手机号 | 必须为合法手机号 |
| contactRelation | enum | 必填 | 见枚举列表 |

联系人关系枚举（建议）：
- 父母
- 配偶
- 子女
- 同事
- 朋友
- 其他

### 3) 工作情况
| 字段 | 类型 | 规则 | 说明 |
| --- | --- | --- | --- |
| employmentType | enum | 必填 | 见枚举列表 |
| employmentName | string | 2-50 字 | 单位名称或职业名称；无业/学生/退休时可填“无” |

工作类型枚举（建议）：
- 全职
- 兼职
- 自由职业
- 个体经营
- 无业
- 学生
- 退休

### 4) 收入情况
| 字段 | 类型 | 规则 | 说明 |
| --- | --- | --- | --- |
| incomeRange | enum | 必填 | 见枚举列表 |

收入区间枚举（建议）：
- 0-1000
- 1000-2000
- 2001-3000
- 3001-4000
- 4001-5000
- 5001-8000
- 8001-12000
- 12001-20000
- 20000+

### 5) 住家地址
| 字段 | 类型 | 规则 | 说明 |
| --- | --- | --- | --- |
| provinceCode | string | 必填 | 省份编码 |
| cityCode | string | 必填 | 城市编码 |
| districtCode | string | 必填 | 区县编码 |
| provinceName | string | 必填 | 省份名称 |
| cityName | string | 必填 | 城市名称 |
| districtName | string | 必填 | 区县名称 |
| detailAddress | string | 5-100 字 | 详细地址 |

## 交互与校验
- 地址使用“省/市/区”级联选择，选择省后拉取市，选择市后拉取区。
- 省市区与详细地址全部必填。
- 工作情况与收入为选择项，不允许手填自由文本。
- 所有字段必须在提交审核前一次性填写完成，提交后不可修改（若需修改由后台驳回后重填）。

## 接口与数据落地建议
- 在下单资料提交接口中新增以上字段。
- 订单详情接口返回新增字段，供后台审核页展示。
- 保存省/市/区编码与名称，避免后续名称变化导致展示异常。

## 验收标准
- H5 表单必须阻止空提交，校验提示明确。
- 后台订单详情可完整展示新增字段。
- CSV 导出（如有）包含新增字段。
