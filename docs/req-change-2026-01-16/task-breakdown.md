# 开发拆分任务

本任务拆分按“后端 -> 管理端 -> H5 -> 数据/测试”顺序推进。

## 后端（API + 数据）
- 新增/更新下单资料字段（联系人、工作情况、收入、住家地址）。
- 订单资料提交接口校验必填，补充字段落库。
- 订单详情接口返回新增字段（用于后台审核）。
- 新增商品分类表与关联逻辑，支持三级分类。
- 新增分类树查询接口与商品按分类筛选。
- 订单调价接口：仅允许未签署状态，更新金额与租期，生成新合同。
- 新增调价日志表或日志字段。

## 管理端（Admin）
- 订单详情页展示新增资料字段。
- 订单详情增加调价入口与表单校验。
- 商品新增/编辑支持三级分类选择。
- 分类管理页（新增/编辑/排序/启用禁用）。

## H5（用户端）
- 资料补充表单新增联系人、工作、收入、地址。
- 地址级联选择联动省/市/区。
- 资料提交前校验必填项。
- 商品选择流程增加“大类 -> 系列 -> 型号”。
- 订单详情支持显示调价提示与最新金额。

## 数据与配置
- 补全分类基础数据（初始化大类/系列/型号）。
- 收入区间与工作类型枚举统一配置（前后端一致）。

## 测试与验收
- API 字段校验与存储测试。
- 订单调价：允许/不允许时点校验。
- H5 表单与联动地址选择测试。
- 分类选择与订单创建流程回归测试。
