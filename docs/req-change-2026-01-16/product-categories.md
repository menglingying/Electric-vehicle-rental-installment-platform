# 商品分类与选择流程（大类 -> 系列 -> 型号）

## 目标
当前商品选择只有“具体型号”。需要增加“先选大类，再选系列，再选具体型号”的三级结构，优化筛选与展示。

## 推荐方案
### 数据结构
新增商品分类表（三级）：
- `product_category`
  - `id`
  - `name`
  - `level` (1/2/3)
  - `parent_id`
  - `sort`
  - `enabled`

商品表新增关联：
- `category_id`：指向三级分类（型号级）
- 也可冗余保存 `category_level1_id` / `category_level2_id`（便于查询和统计）

### 后台管理
- 商品新增/编辑时：先选大类，再选系列，再选型号。
- 支持分类管理（新增/编辑/排序/启用禁用）。
- 商品列表支持按大类/系列/型号筛选。

### 前端选择流程（H5）
1) 进入商品列表或筛选入口时展示大类。
2) 选择大类后展示其系列。
3) 选择系列后展示该系列下的型号（具体商品）。
4) 进入商品详情并下单。

## 接口建议
- `GET /api/common/product-categories/tree`：返回完整树（一级含二级、三级）。
- `GET /api/products?categoryId=xxx`：按三级分类筛选商品。

## 验收标准
- 后台可维护三级分类并绑定商品。
- H5 流程可按“一级 -> 二级 -> 三级”选择商品。
- 订单中记录用户选择的三级分类（用于审核与统计）。
