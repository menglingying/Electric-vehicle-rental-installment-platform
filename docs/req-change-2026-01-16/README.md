# 需求补充整理（按推荐方案落地）

日期：2026-01-16

本文件夹用于记录本次客户新增需求的整理结果与开发拆分。已经按“推荐方案”做出默认选择，后续开发以此为准。

## 需求摘要
- 下单后补全资料不再仅限身份证，需补充联系人、工作情况、收入情况、住家地址。
- 商品选择需要先展示大分类，再选择系列，最后选择具体型号。
- 订单创建后允许内部调价（租期不同，租金不同），但有明确的时点与合同规则。

## 关键决策（已确定）
- 资料必须在提交审核前一次性填写完成，不允许“下单后分步补录”。可以分步骤引导填写，但最终必须完整提交。
- 地址为“省/市/区 + 详细地址”，全部必填；省市区使用联动选择。
- 收入为区间选择，工作情况为枚举选择，全部必填。
- 调价仅允许在“未签署合同/未生效”阶段进行，调价后必须重新生成合同并重签。
- 商品分类采用三级结构：大类 -> 系列 -> 型号；前端级联选择。
