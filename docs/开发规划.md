# 开发规划（一期：不含通讯录/定位）

## 0. 范围声明（先定边界）

一期明确不做：
- 通讯录：不获取、不展示、不存储
- 定位：不采集、不展示、不存储

一期交付形态：
- **H5 客户端**（移动端优先，可 PC 打开）
- **PC 后台管理端**
- **服务端 API + 管理后台接口**

## 1. 技术栈（建议落地方案）

- H5：Vue 3 + TypeScript + Vite + Vant 4
- PC 后台：Vue 3 + TypeScript + Vite + Arco Design + ArcoDesign Pro（Vue）
- 服务端：Spring Boot 3 + MySQL + Redis
- 文件存储：MinIO（本地/私有化）或 OSS（云）
- 定时任务：XXL-JOB（推荐）或 Quartz（简单场景）
- 第三方：电子签（腾讯电子签/法大大/上上签）；短信（可选）；支付（可选）

UI 规范：
- `docs/UI_视觉与交互要求.md`
- `docs/UI_UIX_Pro_Max_执行规范.md`

仓库结构（已落地）：
- `apps/h5`：H5 客户端（Vue3 + Vant）
- `apps/admin`：PC 后台（Vue3 + Arco）
- `services/api`：服务端（Spring Boot 3）
- `services/api-node`：服务端（Node/Express，便于本地联调；后续可替换为 Spring Boot 持久化版本）

## 2. 模块拆分（按业务域）

### 2.1 H5 客户端（客户侧）
- 商品：列表、详情
- 下单：手机号登录（待定）、资料填写、期数/周期/押金方案选择、确认页
- 订单：列表、详情（含合同状态、还款计划、还款记录）
- 电子签：发起签署、签署结果展示（跳转/回跳）

### 2.2 PC 后台（运营/审核/财务/催收/管理员）
- 登录与权限：账号、角色、菜单权限、数据范围（最小可用）
- 商品管理：上架/下架、租金与配置、图片
- 订单管理：列表、详情、审核、状态流转
- 还款管理：还款计划、收款记录、对账导出
- 提醒与逾期：到期前提醒列表、逾期分层列表（1-3 / 3-10 / 10-30 / 30+）
- 黑名单：维护、留痕、前端拦截提示
- 系统配置：期数/周期/押金比例/押金分期方案字典
- 操作审计：关键操作留痕（黑名单变更、订单审核、还款调整、第三方配置）

### 2.3 服务端（API + 业务引擎）
- 认证与授权：H5 登录态、后台 RBAC、接口鉴权
- 核心计算：租金分期计划生成、押金方案与抵扣规则（待确认）
- 订单状态机：下单→签署→审核→生效→结清/关闭（具体待确认）
- 电子签对接：签署发起、回调验签、合同 PDF 存档、订单状态更新
- 提醒与逾期：到期任务生成、逾期天数计算、分层查询
- 文件与资源：图片/合同 PDF 上传、下载权限控制

## 3. 一期里程碑（可按周/迭代执行）

### M0：项目初始化（0.5–1 天）
- 创建仓库结构（H5、后台、服务端三部分）
- 统一环境：Node.js、pnpm/npm、Java 17、MySQL、Redis、MinIO
- CI（可选）：构建与基本校验

### M1：基础能力（1–2 周）
- 服务端：用户/角色/权限、商品、订单骨架、字典配置、审计日志
- 后台：登录、菜单、商品管理（可用）
- H5：商品列表/详情（可用）

### M2：核心下单与计划（1–2 周）
- H5：下单表单、期数/周期/押金方案选择、确认页、订单列表/详情
- 服务端：计划生成、订单创建、订单查询、黑名单拦截
- 后台：订单列表/详情、审核入口（若一期需要审核）

### M3：电子签跑通（1–2 周，依赖第三方资料）
- 服务端：发起签署 + 回调处理 + 合同存档 + 订单状态更新
- H5：跳转签署页/回跳结果页、合同预览/下载入口
- 后台：订单签署状态可视、合同下载

### M4：提醒与逾期分层（0.5–1 周）
- 服务端：到期前 3 天/当天提醒任务、逾期天数与分层
- 后台：提醒列表、逾期分层列表（1-3/3-10/10-30/30+）
- 短信（可选）：只做触发与记录，不做“必须送达”

### M5：验收与交付（0.5–1 周）
- 验收用例回归（按 `docs/需求文档` 第 11 章）
- 异常兜底与幂等：电子签回调、重复回调、网络失败重试
- 数据导出（财务/催收最小可用）

## 4. 首批数据模型（MVP 版）

建议先落这些表（字段后续按细化补充）：
- 用户/角色/权限：`admin_user`、`admin_role`、`admin_permission`、`admin_audit_log`
- 商品：`product`、`product_image`
- 订单：`order`、`order_item`、`order_status_log`
- 还款：`repayment_plan`、`repayment_record`
- 黑名单：`blacklist`
- 字典配置：`dict`、`dict_item`
- 电子签：`contract`（第三方单号、状态、pdf_key、回调原文、验签结果）

## 5. API 清单（按优先级）

H5：
- 商品：`GET /api/h5/products`、`GET /api/h5/products/{id}`
- 下单：`POST /api/h5/orders`
- 订单：`GET /api/h5/orders`、`GET /api/h5/orders/{id}`
- 电子签：`POST /api/h5/contracts/{orderId}/start`、`GET /api/h5/contracts/{orderId}`

后台：
- 登录：`POST /api/admin/auth/login`
- 商品：`/api/admin/products/*`
- 订单：`/api/admin/orders/*`（含审核动作）
- 还款：`/api/admin/repayments/*`
- 提醒/逾期：`/api/admin/reminders/*`、`/api/admin/overdue/*`
- 黑名单：`/api/admin/blacklist/*`
- 字典：`/api/admin/dicts/*`
- 审计：`GET /api/admin/audit-logs`

## 6. 已确认事项（一期执行口径）

1) 登录方式：手机号验证码登录；一期先用“固定验证码”用于联调与测试（必须做环境开关，生产必须接入短信并移除固定码）。
2) 订单状态：一期需要人工审核（审核通过后订单生效）。
3) 收款方案：后续对接第三方 H5 收款/收银台（一期先预留接口与状态流转）。
4) 押金抵扣规则：按 `docs/需求文档` 第 8 章口径落地。
5) 电子签供应商：一期先预留对接能力（发起、回调、合同存档、订单状态更新），供应商后续确定再接入。

## 7. 仍需确认的问题（会影响排期与实现细节）

1) 交付/取车/归还等履约状态：是否纳入一期？若不纳入，订单“生效后”到“结清/关闭”的状态如何定义。
2) 支付回调：是否需要“支付成功回调自动入账 + 对账”，还是一期仅做“跳转收银台 + 人工登记/导入”。
3) 押金：是否可退、退还规则、抵扣最后一期的精确实现细节（是否允许跨多期抵扣）。
4) 电子签：最终供应商、API 文档、回调验签方式、测试环境账号与签署流程限制。
